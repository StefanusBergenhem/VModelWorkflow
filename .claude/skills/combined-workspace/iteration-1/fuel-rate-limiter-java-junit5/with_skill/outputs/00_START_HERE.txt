FREQUENTLY ASKED QUESTIONS
===========================

Q: Are all tests supposed to pass?
A: Yes. All 40 tests should pass with zero failures or errors.

Q: Can I modify the implementation?
A: Yes. Any change that breaks a test is a regression. See COVERAGE_MATRIX.md
   to understand what each test validates.

Q: What JDK version is required?
A: Java 17 or newer (matches design specification).

Q: How long do tests take to run?
A: < 150ms total (most tests < 10ms, concurrency test < 100ms).

Q: Why is the code so simple?
A: By design. Complexity limits (50 lines, CC < 10) ensure maintainability.
   See IMPLEMENTATION_NOTES.md for rationale.

Q: Is this production-ready?
A: Yes. All design elements are implemented, tested, and documented.
   See DELIVERY_MANIFEST.md for sign-off checklist.

Q: Can I use this in my project?
A: Yes. Standard Maven package structure, standard JUnit 5 tests, no external
   dependencies (except JUnit). Copy the Java files and build.

Q: What about performance?
A: Constant-time (no allocations, no unbounded loops). Suitable for 1kHz
   control loops. No performance benchmarks included (not in design scope).

Q: Are there any limitations?
A: Configuration is hardcoded (required for determinism). No logging or
   persistence (required for real-time, no I/O). See IMPLEMENTATION_NOTES.md
   section "Known Limitations".
